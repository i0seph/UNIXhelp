경로 목록

전형적인 PATH 환경 변수 값은 다음과 같습니다:

   /bin:/usr/bin:/usr/local/utils/bin:$HOME/bin

경로이름을 구분하는 문자는 콜론(:)입니다. 사용자가 명령행에서 명령어를 입력하면, 쉘에서는 이렇게 구분된 경로를들 먼저 지정한 경로 순서대로 찾아서 실행합니다.

즉, 위에서처럼 환경 변수가 지정되어 있다면, /bin 경로안에도 명령어가 있고, /usr/bin 경로안에도 같은 명령어가 있을때, /bin 경로 안에 있는 것을 실행하게 됩니다.

만약에 지정한 명령어가 이들 경로 안에 없을 경우에는 다음과 같은 메시지를 보여줍니다.

   command_name: command not found

(명심해야 할 것은 유닉스 시스템은 명령어를 실행시킬 때, 그 명령어가 현재 경로 안에 있는지를 검사하지 않습니다. 오직 PATH 환경 변수값으로 지정된 이 경로 목록에서만 찾습니다. 그래서, 이 경로 목록에 포함되어 있지 않는 명령어를 실행시키고자 한다면, 받드시 해당 경로를 먼저 적어주어야 합니다.)
